# Минимальное ориентированное остовное дерево

*(Minimum Cost Arborescence / Directed MST)*

## Описание

В проекте реализованы алгоритмы поиска **минимального по стоимости ориентированного остовного дерева** (arborescence) в ориентированном графе.

Дан ориентированный граф (G = (V, E)), корень (r) и веса рёбер. Требуется найти набор рёбер минимальной суммарной стоимости такой, что:

* каждая вершина (кроме (r)) имеет ровно одного родителя;
* из корня достижимы все вершины.

## Реализованные алгоритмы

1. **Наивная реализация (Chu–Liu/Edmonds)**
   Сложность: (O(VE))

2. **Алгоритм Тарджана (заготовка)**
   Теоретическая сложность: (O(E \log V))

3. **Алгоритм Gabow–Galil–Spencer–Tarjan (заготовка)**
   Теоретическая сложность: (O(E + V \log V))

## Идея алгоритма

Алгоритм Чу–Лю/Эдмондса работает следующим образом:

1. Для каждой вершины выбирается минимальное входящее ребро.
2. Если циклов нет — получено решение.
3. Если есть циклы — они сжимаются в супер-вершины.
4. Рекурсивно решается задача для сжатого графа, затем выполняется восстановление.

В реализации алгоритм применяется к обращённому графу, чтобы получить дерево, достижимое из корня.

## Структура проекта

* `include/` — заголовочные файлы (API)
* `src/` — реализация алгоритмов и `main.c`
* `tests/` — тесты
* `Makefile` — сборка проекта

## Сборка и запуск

```bash
make
make test
```

Или вручную:

```bash
gcc -std=c99 -Wall -Wextra -Iinclude src/*.c -o dmst
./dmst [naive|tarjan|gabow] < input.txt
```

## Тесты

Проверяются:

* тривиальные графы
* несвязные графы
* графы с циклами
* отрицательные веса
* кратные рёбра
